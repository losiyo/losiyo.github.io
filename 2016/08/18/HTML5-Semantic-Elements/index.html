HTML5语义化元素

发表于 2016-08-18   |   分类于 HTML5

< section >

  • <section>标签定义文档中的节。比如章节，页眉，页脚或文档中的其余部分

< article >

  • 定义独立的，完整的内容
  • <article>标签的内容独立于文档的其余部分

< header >

  • 网页或”section”的页眉
  • 没有个数限制

< footer >

  • 定义文档或section的页脚
  • 应该包含它的包含元素
  • 一个页脚通常包含文档的作者，著作权信息，联系信息，网站地图，回到顶部链接，相关文件
  • 一个文档可以包含多个footer元素

< nav >

  • 定义导航链接的部分
  • 不是所有的链接都要包含在nav标签中

< aside >

  • 定义页面区域内容之外的内容
  • aside标签的内容应与主域的内容相关

< main >

  • IE 11- 不支持
  • 呈现文档<body>或应用的主体部分。主体部分由与文档直接相关，或者扩展于文档的中心主题，应用的主要功能部分组成。这部分内容在文档中应该是独一无二的，不包含在任何一系列文档中重复的内容，比如侧边栏，导航栏链接，版权信息，网站logo，搜索框。

< figure > < figcaption >

  • <figure>标签规定独立的流内容（图像，图表，照片，代码等等）
  • <figure>元素的内容应该与主内容相关，但如果被删除，则不应对文档流产生影响
  • <figcaption>标签定义<figure>元素的标题
  • <figcaption>元素应该被置于”figure”元素的第一个或最后一个子元素的位置

< details > < summary >

  • IE 和 Edge, safiri 5- 不支持
  • 允许用户创建一个可展开折叠的元件，让一段文字或标题包含一些隐藏的信息

< mark >

  • 在一个引用或者块级元素中，突出显示的文本通常是外部引用的文本，或者
  • 突出显示的文本通常与用户当前活动具有关联性，比如搜索的结果

< time >

  • 表示24小时制时间或公立日期，若表示时间则也可包含时间和时区

浏览器支持

  • 现代浏览器均支持HTML5
  • 所有浏览器都将不认识的元素当作行内元素处理
  • 你甚至可以教会老式浏览器处理不认识的HTML元素

  • HTML5定义了新的语义元素，它们都是块级元素，为了在旧的浏览器正确显示，我们将它们的CSS样式的”display”属性设置为”block”
